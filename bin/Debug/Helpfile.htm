Instructions for SkyFlats SGP

SkyFlats SGP automates the capture of sky flats, by connecting and controlling SGP through its API. The application is particularly targeted towards capturing so-called “blue sky” flats, just before and around sunset, which is possible with the short exposure times of CCD/CMOS cameras without mechanical shutters. SkyFlats SGP dynamically changes the exposure times to reach the targeted ADU, within limits set by the user. The Flat Sequencer handles different filters as needed, binning, etc., and it controls the mount to slew to different possible flat capture positions. The flat sequencer is highly automatized, with the intent of being a “click and forget” application for capturing sky flats.

Before starting the Flat Sequencer, the user must supply correct information in the Setup Form (see below). The application uses SGP (through its API) to take the flat frames, so SGP must be started and the required equipment must be connected in SGP. The Status box should state CONNECTED for all devices before starting the Flat Sequencer.

The mount must be connected directly through an ASCOM driver (i.e. SkyFlats SGP does not support non ASCOM mounts). The Mount box allows to connect to the mount and to manually slew to the flat capture position, as defined in the Setup Form below. Note that the ALT, AZ, RA, and DEC values are pulled directly from the ASCOM driver, hence they do not depend on the location settings in the Setup form. Note that the mount is automatically slewed to the flat position when starting the Flat Sequencer. The Slew button in the Mount box allows to independently check that the mount slews to the desired position. PLEASE READ THE DISCLAIMER BELOW BEFORE ATEMPTING TO SLEW THE MOUNT!!

 

The Flat Sequencer box is the core of the application. The Setup button opens the Setup Form, where parameters about the User Location, Mount, Camera and Filter settings must be properly entered before attempting to run the Flat Sequencer.

!!!! Some ASCOM mounts do not allow to slew to ALT and AZ coordinates, this is the case for EQMOD for instance. In such a case SkyFlats SGP uses the Location settings to calculate the RA and DEC coordinates corresponding to the ALT and AZ coordinates. Failure to correctly configure the Location data can result in slews of the telescope to dangerous positions.
DISCLAIMER: THE AUTHOR OF THE APPLICATION TAKES NO RESPONSIBILTY IN CASE OF DAMAGE OF EQUIPMENT. THE USER MUST PROVIDE PROPER LOCATION DATA AND CONFIGURE THE SAFETY LIMITS OF THE ASCOM DRIVER/MOUNT TO AVOID COLLISIONS DUE TO POTENTIAL SLEWS OUTSIDE THE SAFE SLEW ZONE. !!!!

When the Location data has been properly set in the Setup Form, the Flat sequencer box will show the Time and Azimuth of the Sunset at the user Location. The Start Time is configured by setting the proper start time delay in the General Settings box in the Setup Form. In this example the delay is set to -40 min, so that the Flat Sequencer will start at 19.27 as the sunset is at 20.07.
The Start Delay shows the time until the Start time, with respect to the current time. If the Start Time has already passed it shows “Start Now”.

The Flat Sequencer is executed by clicking the Start button. The user will be prompted to confirm the slew to the flat capture position defined in the Setup Form. After the slew, the telescope will stop tracking if the corresponding checkbox has been checked in the Setup Form. The application will then wait until the Start time before actually starting the Flat Sequencer.



The Flat Status box shows a summary of the progress of the Flat Sequencer. More detailed information can be found in the Log Window, which can be shown by clicking on the Show Log checkbox. A logfile is automatically created each time SkyFlats SGP is started up. The logfile can be found under \Users\USERNAME\AppData\Roaming\SkyFlatsSGP (the same directoy is used to store the settings file settings.json).

 

Setup Form

General settings

Save directory – Output directory for flat fit files produced by the Flat Sequencer. Files will be put in sub-folders with the filter names, and the flat files names will contain information on exposure time, binning etc.

Location – User location Latitude and Longitude in the units of degrees (fractional degrees). These are used to correctly calculate sunset times, azimuth of sunset, and convert ALT,AZ coordinates to RA,DEC for the telescope slew. PUTTING WRONG VALUES HERE CAN SEND THE TELESCOPE TO AN UNEXPECTED POSITION !!

Start time – flat sequence start time with respect to sunset (negative value = start X minutes before sunset, positive value = start X minutes after sunset). The sunset time, actual start time, and the delay to the start time are shown in the main form. If the start time is already passed, then the delay says “Start Now” in the main form. When the Flat Sequencer is started, the mount will first slew to its flat position, and (if necessary) it will wait for the start time.

 

Mount settings

ASCOM mounts – Dropdown menu with available ASCOM mounts. This is an ASCOM mount only application.

SkyFlats SGP supports four different flat positions:

·       Slew to preset position – Slews to ALT and AZ coordinates (both in degrees). This is useful if the zenit position is not possible to reach due to obstacles etc.

·       Slew to “minimum gradient” position – Slews to 15 degress below Zenith to the east (anti-solar) on the solar circle at sunset (i.e. the azimuth angle at unset). This is a traditional sky flat position used by professional observatories (not implemented yet).

·       Slew to Zenith

·       Park mount – Parks to the ASCOM predefined park position (note that for EQMOD users this is NOT any of the park positions defined within EQMOD, but the same position as defined within SGP which also uses the ASCOM park position)

Stop TRACKING during capture – When checked the mounts stops tracking after having slewed to the flat position. If there is a start delay, then the mount stops tracking and then waits. Tracking resumes after the Flat sequencer has stopped.

 

Camera settings

Min exposure time – Minimum exposure time (in unit of seconds) for the Flat sequencer, must be longer than the minimum allowed camera exposure time. In the flat calibration process (per filter), an estimated exposure time “exptime” required to reach the target ADU will be calculated based on the calibration flat frames. If “exptime” is below the minimum exposure time, meaning that the sky is too bright, then the process will pause 60 seconds and take another flat calibration frame until “exptime” is above this value.

Max exposure time – Maximum exposure time (in unit of seconds) for the Flat sequencer. If the “exptime” is longer than this value the Flat sequencer will abort completely, as the sky is too dark.

Min and Max ADU – The goal of the application is to reach a target ADU for each flat frame, which is calculated as the mean of the Min and Max ADU (target ADU= (MinADU+MaxADU)/2). During the calibration process a new calibration flat frame will be captured as long as the mean ADU of the calibration frame is below Min ADU.

Calibration frame size – To speed up the exposure calibration process (per filter), the calibration frame can be taken at a reduced frame size, which greatly reduces download speeds. If possible, use Third or Quarter frame sizes. Note that the reduced frames are centered on the full frame image.

Flat frames – Number of flat frames to be recorded by the Flat sequencer, per each filter.

Auto adjust exposure time per frame – The exposure time calibration process is done before starting the flat frame exposures, for each filter. However, since the sky brightness is also decreasing during the capture of the actual flat frames, to reach the target ADU for each frame the exposure time can by dynamically adjusted by checking this box.

Filter settings

The Flat Sequencer will capture flat frames for a list of filters as defined in the Filter settings. The Flat Sequencer will run through the list from top to bottom. Filters can be activated/disactivated by clicking on the Use checkbox before each filter in the list. The position of a filter can be changed by selecting the corresponding line in the filter list and pressing the Up and Down buttons.

The order of the filters is particularly important when taking “blue sky” flats. For instance, as it takes quite some time to record many 1x1 binned flat frames for the narrowband filters, these can be recorded when the sky is much too bright for capturing LRGB flat frames even with the shortest exposure times of modern cameras. One should also consider that the sky is brighter for filters towards the blue, while red filters see a darker sky. The list below is probably a good starting point for a combination of narrowband filters (binned 1x1) and Lum (binned 1x1) and RGB (binned 2x2) filters. The order can be fine adjusted by studying the calibrated exposure times found in the log file.

Note that the filter settings are completely independent of the filter settings in SGP. The user must therefore enter each filter by clicking the Add button, where the user must properly enter the correct Position of the filter in the filter wheel. The Add Filter form also allows to select the Binning of the flat frame.

Note that the name of the filter is independent of the name chosen in SGP, but it is convenient to choose the same names as defined in SGP. In SGP the filter settings can be found in the Control Panel, the Filter Tab, by clicking on “Define Filter List”.
